Downloads Router Configuration

Hi there! This extension lets you specify "routing rules" for files that you download.

If a route exists, a suggestion is sent to your browser as it tries to figure out where to save the file. Some items are downloaded directly, but if you right-click and choose "Save As…", your manual selection naturally takes precedence.

You may also define download routes based on the source website. Should a match be made against such a rule, it is considered of higher priority than the filetype-based routes.

Limitations

  • All destination folders must be located within your main Downloads folder
  • Trailing characters for destination folders are essential!
    • Mac OS X and Linux: /
    • Windows: \
  • Currently no dialog-based folder chooser available
    (<input type="file" webkitdirectory> unstable…)

Source domainDestination folder

* Please note that larger websites often utilize Content Distribution Networks, which means that the referrer for a certain image might not be the same as the domain of the website you are visiting. Facebook, for instance, uses the Akamai network to serve images.

Common MIME types

This table contains a few common MIME types that might help you set up your routing configuration. A more extensive list is available on Wikipedia.

application/pdfPDF documents
application/zipCompressed archives
application/x-bittorrentTorrent files
audio/mpegMP3 and other MPEG audio
image/gifGIF images
image/jpegJPEG images
image/pngPNG images
video/x-msvideoAVI video
